# 图书管理页面

## 概述

图书管理主页面是系统的核心功能页面，提供完整的图书 CRUD 操作界面。

## 功能特性

### 1. 页面布局
- 使用 Ant Design 的 Layout 组件构建响应式布局
- 顶部导航栏显示系统标题、用户信息和操作按钮
- 主内容区域展示图书列表

### 2. 图书列表展示（需求 4.1-4.6）
- 自动加载并显示所有图书
- 支持加载状态指示器
- 空状态友好提示
- 响应式网格布局（移动端 1 列，平板 2 列，桌面 3-4 列）

### 3. 添加图书（需求 5.1-5.6）
- 点击"添加图书"按钮打开弹窗
- 填写图书信息（名称、作者、描述、封面）
- 表单验证和错误提示
- 成功后刷新列表并显示提示

### 4. 编辑图书（需求 6.1-6.6）
- 点击图书卡片的编辑按钮
- 弹窗预填充当前图书信息
- 修改后提交更新
- 成功后刷新列表并显示提示

### 5. 删除图书（需求 7.1-7.5）
- 点击图书卡片的删除按钮
- 显示确认对话框
- 确认后删除并刷新列表
- 显示操作结果提示

### 6. 用户登出（需求 8.1-8.3）
- 点击登出按钮
- 清除认证令牌
- 重定向到登录页面

## 组件结构

```
BookManagementPage
├── Header
│   ├── 系统标题
│   ├── 用户信息
│   ├── 添加图书按钮
│   └── 登出按钮
├── Content
│   └── BookList
│       └── BookCard (多个)
└── BookFormModal (弹窗)
```

## 状态管理

### 本地状态
- `isModalVisible`: 控制弹窗显示/隐藏
- `editingBook`: 当前正在编辑的图书（undefined 表示添加模式）

### Hook 状态
- `useBooks`: 管理图书列表和 CRUD 操作
- `useAuth`: 管理用户认证状态

## 响应式设计

### 桌面端（> 768px）
- 水平布局的头部
- 图书列表 3-4 列网格

### 平板端（576px - 768px）
- 头部元素垂直排列
- 图书列表 2 列网格

### 移动端（< 576px）
- 紧凑的头部布局
- 图书列表 1 列网格
- 按钮自适应宽度

## 用户交互流程

### 添加图书流程
1. 用户点击"添加图书"按钮
2. 打开空白表单弹窗
3. 填写图书信息
4. 上传封面图片（可选）
5. 点击提交
6. 显示成功提示，关闭弹窗，刷新列表

### 编辑图书流程
1. 用户点击图书卡片的编辑按钮
2. 打开预填充数据的表单弹窗
3. 修改图书信息
4. 点击提交
5. 显示成功提示，关闭弹窗，刷新列表

### 删除图书流程
1. 用户点击图书卡片的删除按钮
2. 显示确认对话框
3. 用户确认删除
4. 执行删除操作
5. 显示成功提示，刷新列表

## 错误处理

- API 请求失败时显示错误提示
- 表单验证失败时显示验证错误
- 网络错误时提供友好的错误信息

## 依赖组件

- `BookList`: 图书列表组件
- `BookCard`: 图书卡片组件
- `BookFormModal`: 图书表单弹窗组件
- `useBooks`: 图书管理 Hook
- `useAuth`: 认证管理 Hook
